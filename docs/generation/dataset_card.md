# Phase 6 — Generation dataset card

Structured-input -> Findings/Impression training data, built from PTB-XL by `scripts/build_gen_dataset.py`. Regenerate with `make gen-data`.

## Composition

- **train**: 11,675 examples
- **val**: 2,183 examples
- **test**: 2,198 examples

Filtered to `validated_by_human == True` records only, so every target report reflects an actual cardiologist's confirmed label set, not an autogenerated one. Splits follow PTB-XL's official `strat_fold` — the same folds `src/detection/train.py` uses — so a generator trained here never sees a record the detector was evaluated on.

## Fields (per JSONL row)

`ecg_id, codes, age, sex, heart_rate_bpm, heart_axis, confidences, leads, review_threshold, original_report, findings, impression`. `heart_rate_bpm` is **measured** — Pan-Tompkins R-peaks run on the record's own Phase-2-preprocessed signal — not templated from the rhythm label. `original_report` is PTB-XL's own (German, autogenerated-then-cardiologist-validated) free-text report, carried through for provenance/spot-checking, not used as a training target. `confidences` are synthetically sampled per finding (mostly 0.82-0.99, ~15% sampled 0.30-0.49) so the model sees the "(requires clinician confirmation)" phrasing during training; PTB-XL labels carry no native confidence signal.

## Known caveats

- Target text is **template-rendered**, not human-authored — it teaches the generator the *exact* two-section format and a controlled clinical vocabulary, but its prose is more formulaic than a cardiologist's. See `docs/generation/examples_review.md` for a reviewed comparison against PTB-XL's own human report text.
- `heart_axis` is only surfaced when unambiguous (LAD/RAD and extreme forms); orientation codes of uncertain clinical framing (AXL/AXR/SAG) and the normal bucket (MID) are recorded but not templated into the report text.
- Confidence sampling is synthetic — a real detector's calibration differs (see `docs/baseline/`); the generator should be re-checked against live detector confidence distributions before production use.
